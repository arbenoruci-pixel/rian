AUDIT — PRANIMI / “TË PA PLOTËSUARA” — V5

Finding:
V4 blocked stale remote drafts in fetch/filter/open/delete paths, but React could still render old draft state before refresh completed, or render remote summaries that did not carry a proof that they had passed the latest DB guard.

Important distinction:
- Storage stale objects can exist and must not be trusted.
- The UI state must also be treated as untrusted.
- Therefore, the render path itself needs a final guard, not only the data-fetch path.

V5 keeps the same DB/Storage policy and adds a last-mile render gate:
- Remote summaries must have _draft_ui_guard = tepaplotesuara-v5-verified-before-render.
- That marker is added only after filterDraftSummariesAgainstDb() completes.
- uniqueDrafts and pushDraftsToState both use shouldDraftSummaryRender().
- openDrafts clears current list and cache before loading fresh verified rows.

No other module was touched.
