AUDIT — PRANIMI / BAZ / “TE PAPLOTESUARA”
Patch: PRANIMI_TEPAPLOTESUARA_SAFE_CODE_LIFECYCLE_V1

Files/functions that control the current flow:

1) app/pranimi/page.jsx
   - resetForNewOrder()
     Opens a fresh PRANIMI session, creates a new local_oid, clears stale UI state, and schedules code reservation through tryReserveCodeInBackground().

   - tryReserveCodeInBackground(nextOid, reason)
     Gets/displays the BAZ code for the active local_oid via reserveSharedCode(). Before this patch it did not persist an explicit draft/code lifecycle record. The patch records a local reservation immediately and sets/refreshes a short DB lease for blank draft sessions.

   - buildDraftSnapshot()
     Builds the draft payload from form state. The patch adds local_oid, code, created_by_pin, created_at, last_activity_at, has_meaningful_work, and draft_lifecycle.

   - snapshotHasMeaningfulWork()
     Decides whether a draft is real. Before this patch, name-only or phone-only work could be ignored. The patch counts name and phone as meaningful work, along with notes/photos/m2/payment/items.

   - upsertDraftLocal(), readAllDraftsLocal(), removeDraftLocal()
     Manage local draft mirrors in localStorage under draft_orders_v1 and draft_order_<id>. Before this patch, local draft helpers existed, but the normal autosave path mainly uploaded remote drafts. The patch writes local draft mirrors only for meaningful work.

   - upsertDraftRemote(), readDraftRemote(), listDraftsRemote(), fetchRemoteDraftsSummary()
     Manage remote draft JSON in Supabase Storage under tepiha-photos/drafts. The patch filters out blank drafts and requires a real code.

   - refreshDrafts(), pushDraftsToState(), uniqueDrafts
     Build the “TE PAPLOTESUARA” list. Before this patch the list could rely on remote cache only, clear local mirrors after remote success, and dedupe by code. The patch merges local + remote, filters against public.orders, and dedupes by local_oid/id.

   - applyDraftSnapshotToForm(), loadDraftIntoForm()
     Opens a draft. Before this patch, opening a draft could keep the active/stale local_oid/code and ignore the draft’s stored code. The patch restores the exact draft local_oid and code.

   - handleContinue() and finishSuccess()
     Finalize the order. The previous patch already made DB order verification the success gate. This patch removes the draft reservation only after finishSuccess(), which is reached after DB verify.

   - deleteDraft(id)
     Deletes an incomplete draft. Before this patch it removed the draft UI/storage, but did not release the code pool lock. The patch releases the code when no order exists and clears only that draft reservation.

2) lib/baseCodes.js
   - reserveSharedCode(oid)
     Gets a code from the existing base code pool and stores base_order_code:<oid>.

   - markCodeUsed(code, oid)
     Marks a code used after order save/verify.

   - releaseLocksForCode(code, opts)
     Was previously a compatibility no-op. The patch makes it release only a DB reservation owned by this worker and never release a code that already has an order.

   - holdBaseCodeForDraft(code, oid, opts)
     Added by this patch. It refreshes/holds base_code_pool reservation metadata for a draft without schema change.

Parallel flows found:

- Local draft mirror: localStorage draft_orders_v1 + draft_order_<local_oid>.
- Remote draft mirror: Supabase Storage JSON under drafts/<local_oid>.json.
- Active session mirror: tepiha_pranimi_current_session_v1.
- Code reservation mirror: base_order_code:<local_oid> from lib/baseCodes.js.
- Code pool DB lifecycle: base_code_pool status/reserved_by/reserved_at/lease_expires_at/status used.
- Final order path: public.orders save + DB verify + markCodeUsed().
- Offline/outbox order path: local/outbox save with LOCAL / NOT SYNCED warning, no final success without DB verify.

Main risks found before patch:

- Blank sessions had no explicit PRANIMI draft lifecycle record to tell blank reservation from meaningful draft.
- Name-only or phone-only draft work could be missed by meaningful-work detection.
- Opening a draft could ignore the draft code and keep stale active code.
- The “TE PAPLOTESUARA” list could show stale/remote cache entries after an order existed in public.orders.
- Deleting a draft did not release the code pool reservation.
- Local unsynced drafts could be hidden when remote draft fetch succeeded.
