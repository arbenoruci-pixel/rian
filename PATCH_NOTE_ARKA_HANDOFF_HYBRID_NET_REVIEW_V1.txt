PATCH: ARKA / Dispatch handoff review hybrid double-commission fix

Touched files:
- app/arka/page.jsx

Scope:
- UI/review calculation only for ARKA Dispatch handoff accept modal.
- No DB changes.
- No SQL/RPC changes.
- No payroll/company budget changes.
- No Service Worker/cache/runtime changes.

Bug fixed:
- cash_handoff_items.amount is already NET / PËR BAZË for hybrid transport handoffs.
- The review modal was treating item amounts as gross when sum(item.amount) matched cash_handoffs.amount, then subtracting hybrid commission again.
- Example wrong path: 591.76 - 211.80 = 379.96.

Correct behavior:
- For stored cash_handoff_items, amount is treated as base/net.
- Commission is calculated from transport_m2 * worker commission rate for display.
- Gross display uses pending payment gross if available; otherwise falls back to item.amount + commission.
- cash_handoffs.amount is compared against sum cash_handoff_items.amount.
- Legacy gross inference is only allowed when raw item total minus commission matches the handoff amount.

Expected for handoff #135:
- Gross/client paid display: about 803.56 EUR when transport m2 metadata is present.
- Commission display: about 211.80 EUR.
- Per baze display: 591.76 EUR.
- No false warning showing recalculated per baze 379.96 EUR.
