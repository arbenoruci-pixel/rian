PATCH NOTE — TRANSPORT DUPLICATE / CLIENT-MIX GUARDS V1

Scope:
- lib/transport/transportDb.js
- lib/transportCodes.js
- app/dispatch/page.jsx

Purpose:
Prevent the production issues observed with T201/T136/T257:
1) a stale client_id/T-code reused for a different customer,
2) a T-code marked available while an order/client already uses it,
3) dispatch-created transport orders not marking the reserved T-code as used.

Changes:
- Transport client upsert now refuses to update an existing transport_clients row by id if the incoming phone does not match the current client's phone.
- T-code fallback now refuses to reuse a tcode row if that T-code already belongs to another phone/customer.
- Transport code reservation now verifies candidate codes against transport_orders and transport_clients before returning them online.
- Cached draft T-codes are also verified online; if already used, they are cleared and a new safe code is requested.
- markTransportCodeUsed now also stores owner_id with the user/driver pin.
- Dispatch marks a T-code as used after creating a transport order.

Not touched:
- DB schema
- payroll/commission rules
- Service Worker/cache/runtime
- offlineStore/outbox format
- order status flow
- ARKA cash handoff logic
