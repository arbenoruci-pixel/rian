PATCH NOTE - PRANIMI / TE PA PLOTESUARA - LOCAL EXIT SAVE V6

Scope:
- app/pranimi/page.jsx only
- No DB schema/RPC changes
- No Storage mass cleanup
- No transport/arka/pastrim/gati/dispatch changes

Reason:
V5 fixed stale remote Storage drafts, but it over-tightened the render/list gate for newly typed local drafts.
A user could type meaningful info (name/phone/etc.), tap HOME quickly, and the draft could fail to appear in TE PA PLOTESUARA because:
- autosave timer could be cancelled before firing,
- remote upload/code hold could lag,
- code_pool guard could see the code as available before the local meaningful draft lease had caught up and then hide/remove the local draft.

Changes:
1. Added immediate local-first draft persistence before any network work:
   - persistMeaningfulDraft() now writes local draft + current session before awaiting Supabase Storage upload.
   - HOME flow now calls persistCurrentDraftLocalSync() before async remote/code operations.

2. Added exit safety persistence:
   - pagehide
   - beforeunload
   - visibilitychange hidden
   These save meaningful work locally without waiting for remote Storage.

3. HOME button now saves on pointer/mouse down before navigation click:
   - protects very quick taps.

4. Code pool guard no longer deletes a just-created local meaningful draft only because base_code_pool still says available:
   - local draft with local meaningful reservation is allowed temporarily.
   - the app immediately attempts to re-hold the code in base_code_pool.
   - remote Storage drafts remain strict/fail-closed.

Expected behavior:
- Old stale remote drafts like drafts/1306.json stay hidden.
- Typing only a name or phone, then tapping HOME, keeps the draft visible as LOCAL / NOT SYNCED or synced draft.
- Final order success still requires DB verify from previous patch.
